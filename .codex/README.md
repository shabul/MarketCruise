# `.codex/` Project Context

This folder is the Codex-oriented companion to `.claude/`. It is intentionally short and implementation-focused.

## Reading Order

1. `AGENTS.md`
2. `context/quick-reference.md`
3. `context/conventions.md`
4. `context/troubleshooting.md`
5. `context/session-log.md`

## Purpose

- Keep project-local operating context for future Codex sessions.
- Prefer implementation truth over aspirational docs.
- Record only high-signal decisions, traps, and runtime flow details.

## Maintenance

- Update `quick-reference.md` when files move or flows change.
- Update `conventions.md` when a new rule can prevent a real regression.
- Append to `session-log.md` only for decisions or surprises worth carrying forward.
